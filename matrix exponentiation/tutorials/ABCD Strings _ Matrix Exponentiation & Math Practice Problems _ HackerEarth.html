ABCD Strings

Please find the number of N-length strings that are made of only letters 'a, 'b', 'c' and 'd' and do not contain any of the restricted strings.

Since, the answer can be very large, please print it modulo 1000000007.

Input Format:

The first line of input file contains T, the number of test cases to follow.
The first line of each test case contains 2 space-separated integers N and K.
The next K lines each contain a string S, which represents a restricted string.

Output Format:

Output T lines, each containing answer to corresponding test case.

Constraints:

T ≤ 200
N ≤ 1015
K ≤ 50

Problem:
Find the number of N-length strings that are made of only letters 'a, 'b', 'c' and 'd' and do not contain any of the restricted strings.(each of length <= 3)

Idea:

As the constraint on N(upto 1e15) is very high, no trivial solution can pass all tests.

Let's consider an undirected complete graph consisting of a, b, c, d as vertices. All of the strings of length N can be represented as an N length walk on this graph (consider self loops to be present as well).

Now the idea is that, once the proper adjacency matrix is constructed, we can raise to power N, and hence count the number of paths of length N.

Refer Here

Detailed Explanation:

The key to solving this problem is that the length of restricted strings is only upto 3.

Let us consider a case when this length is 1, it would mean that a particular character is disallowed to occur in the string. We can simply disconnect all edges of the corresponding vertex in the graph and implement our idea. Even more simply, this is just (4 - characters_removed) ^ N.

Things begin to get interesting, as we consider a length of 2.
A restricted string ab, means that b should never immediately follow a in any of the strings. Well, our graph analogy is simple and flexible enough to handle this. Simply, remove the edge connectng a and b.

Okay, but how to handle a length of 3?
Suppose that a restricted string is abc, can we disconnect a and b and then b and c, to combat this?

The answer is NO!!

Doing so will mean that ab can also not occur. This will lead us to leaving out on strings like aba, dbc, etc.

Thus, we need to introduce new vertices and a larger graph that contains pairs of vertices each, like ab, bc, etc.

In this new graph, we can handle queries of length 3.

If abc is restricted, consider all vertices having second letter as a, i.e. aa, ba, ca, da and remove their edges to the vertex bc.
At the same time, remove all edges from ab to vertices having c as the first letter, viz. ca, cb, cc, cd.

Note that our graph will have directed edges now, for some eerie implementation details and handling of odd values of N, please refer to the solution(s) below.
